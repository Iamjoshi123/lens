# ADR-003: ORM / Query Builder

**Status**: Accepted
**Date**: 2026-02-26

## Context
Need type-safe database access with migration support. Key requirements: no runtime overhead, TypeScript-first inference, JSONB support, reversible migrations. Evaluated: Drizzle ORM, Prisma, Kysely, raw SQL.

## Decision
**Drizzle ORM with drizzle-kit**

- TypeScript-first: schema is defined in TypeScript; all query results are fully typed without code generation at runtime. No `any` types from query results.
- Lightweight: Drizzle is essentially a typed SQL query builder — no hidden N+1 queries, no lazy loading magic, no active record pattern to work around.
- Migration tooling: `drizzle-kit generate` produces plain SQL migration files that are readable, reviewable, and manually reversible. Down migration SQL files are written alongside each up migration.
- JSONB: Drizzle supports `jsonb()` column type natively for PostgreSQL.

## Consequences
- **Positive**: Full SQL visibility; migration files are plain SQL (easy to review and reverse); zero magic.
- **Negative**: More verbose than Prisma for complex relations; no built-in soft deletes or audit hooks.
- **Trade-off**: Prisma's DX is better for simple CRUD, but Drizzle's explicit SQL approach is better when writing complex scrape deduplication and full-text search queries (Slices 2–3).
